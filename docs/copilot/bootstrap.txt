Summary of the project so far:

- I originally built a Bash watcher that monitored `channels-dvr.log` for `Processing file-` events and triggered a captioning pipeline.
- The Bash version was unreliable due to partial log lines, binary fragments, log rotation issues, AWK crashes, and systemd restart loops.
- I decided to replace it with a Python project with two event sources:
  1. a robust log-based watcher that handles partial lines and rotation,
  2. a ChannelWatch WebSocket backend that provides structured JSON events.
- The Python architecture includes: `watcher`, `log_source`, `channelwatch_source`, `parser`, `state manager`, `pipeline runner`, and `config` system.
- The system tracks last processed timestamps for idempotency and can send optional Apprise notifications.
- The long-term goal is a reliable, extensible, open-source captioning tool for Channels DVR.
