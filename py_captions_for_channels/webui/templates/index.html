    Connect to Channels DVR

    Enter your Channels DVR server URL. The wizard will connect to detect your media folder path automatically.

    Deployment Type

    Where is this captions service running relative to your Channels DVR server?

    Recordings Path

    Confirm the path to your Channels DVR recordings on this machine. It will be bind-mounted into the container at the same path.

    Must be accessible on the Docker host machine.

    Network Mount

    Docker will mount the recordings share directly into the container using the credentials you provide.

    Use forward slashes — e.g. //192.168.3.150/Channels

    The path inside the Docker container where recordings will appear after mounting. You can use any path — /mnt/channels is a good default. The service uses this path to open recording files.

    New Recording Events

    How should the service discover that a new recording is ready to caption?

    How often to check for new recordings. 120 seconds is a good default.

    Timezone of your Channels DVR server. Used to align recording schedules with whitelist time-of-day rules.

    Caption Engine

    Configure the Whisper AI transcription model and how output files are processed.

    The model is downloaded on first use. Larger models require more RAM/VRAM.
